# AI 智能衣橱系统 - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
本项目是一个基于AI技术的智能衣橱管理平台，旨在为用户提供个性化的服装搭配建议、衣橱管理和时尚指导服务。作为毕业设计项目，将综合运用爬虫技术、RAG检索增强生成、Agent工作流、计算机视觉等多种AI技术。

### 1.2 项目定位
- **项目类型**: 个人毕业设计
- **目标用户**: 时尚爱好者、职场人士、穿搭新手
- **开发周期**: 1个月（距答辩）

### 1.3 技术栈
| 层级 | 技术选型 |
|------|----------|
| 后端框架 | Java Spring Boot |
| AI框架 | LangChain4j |
| 大模型 | Qwen3-VL-32B-Instruct（硅基流动 API）|
| 业务数据库 | MySQL 8.0 |
| 向量数据库 | PostgreSQL 15 + pgvector |
| 缓存 | Redis |
| 图像识别 | Qwen3-VL（多模态能力）|
| 天气数据 | 高德 MCP |
| 前端 | Web + 小程序 |

## 2. 功能需求

### 2.1 用户管理模块

#### 2.1.1 用户注册
- **功能描述**: 新用户通过用户名/密码注册账号
- **输入**: 用户名、密码、邮箱
- **输出**: 注册成功/失败提示
- **约束**: 用户名唯一，密码需加密存储

#### 2.1.2 用户登录
- **功能描述**: 已注册用户登录系统
- **输入**: 用户名、密码
- **输出**: JWT Token
- **约束**: 支持Token刷新机制

#### 2.1.3 用户信息管理
- **功能描述**: 用户查看和修改个人信息
- **功能点**:
  - 查看个人资料
  - 修改密码
  - 设置偏好（可选）

### 2.2 衣物管理模块

#### 2.2.1 衣物上传
- **功能描述**: 用户上传自有衣物信息
- **输入**:
  - 衣物照片（必填）
  - 基础信息（可选，AI可自动识别）
- **输出**: 衣物创建成功，返回衣物ID
- **处理流程**:
  1. 接收图片上传，存储到对象存储
  2. 调用AI识别衣物信息（颜色、类别、风格等）
  3. 生成衣物Embedding向量化存储
  4. 保存到数据库

#### 2.2.2 AI自动识别
- **功能描述**: 自动识别衣物图片中的信息
- **识别字段**:
  - 类别: 上装、下装、外套、连衣裙、鞋履、包袋、配饰
  - 颜色: 主色调、辅色调
  - 风格标签: 日系、韩系、欧美、通勤、运动、复古等
  - 材质: 棉、麻、丝、羊毛、化纤、混纺、皮革等
  - 图案: 纯色、条纹、格子、印花等
  - 版型: 修身、宽松、直筒、A字等
  - 适用场合: 日常、通勤、约会、运动、正式、休闲
  - 适用季节: 春、夏、秋、冬
- **技术实现**: 调用Qwen3-VL多模态模型

#### 2.2.3 衣物编辑
- **功能描述**: 修改已上传的衣物信息
- **可编辑字段**: 所有基础信息和AI识别结果
- **约束**: 只能编辑自己的衣物

#### 2.2.4 衣物删除
- **功能描述**: 删除不再需要的衣物记录
- **约束**: 软删除，保留历史记录

#### 2.2.5 衣物列表查询
- **功能描述**: 分页查询用户的所有衣物
- **支持筛选**: 类别、颜色、风格、场合、季节
- **支持排序**: 上传时间、搭配频率、穿着次数

#### 2.2.6 搭配频率设置
- **功能描述**: 为每件衣物设置1-5档搭配频率
- **含义**: 1=很少穿，5=经常穿
- **用途**: 影响推荐排序权重

### 2.4 RAG智能检索模块

#### 2.4.1 向量化存储
- **功能描述**: 将衣物信息转换为向量存储
- **技术**: PostgreSQL + pgvector
- **Embedding模型**: 通过LangChain4j调用

#### 2.4.2 语义检索
- **功能描述**: 基于自然语言查询衣物
- **查询示例**:
  - "找一件适合面试穿的白色衬衫"
  - "推荐几件适合约会的裙子"
  - "我有哪些适合今天天气的外套"
  - "找一件能和这条裤子搭配的上衣"
- **返回**: 按相关度排序的衣物列表

#### 2.4.3 混合检索
- **功能描述**: 结合语义检索和属性筛选
- **支持**: 先语义检索，再按属性过滤

### 2.5 智能搭配建议模块（Agent工作流）

#### 2.5.1 穿搭照片上传
- **功能描述**: 用户上传单张穿搭照片
- **用途**: 获取搭配建议或分析

#### 2.5.2 MCP工具调用
- **功能描述**: 获取实时环境数据
- **工具**: 高德MCP
- **获取数据**:
  - 地理位置
  - 实时天气（温度、天气状况）
  - 季节信息

#### 2.5.3 Agent工作流
**处理流程:**
```
用户上传照片
    ↓
[步骤1: 图像分析]
    ├── 识别服装单品（上衣、下装、外套等）
    ├── 提取颜色信息
    ├── 识别风格特征
    └── 分析场合适配性
    ↓
[步骤2: 环境感知 - MCP调用]
    ├── 获取用户地理位置
    ├── 获取实时天气（温度、天气状况）
    ├── 判断季节
    └── 判断时段（早/中/晚）
    ↓
[步骤3: 知识库检索 - RAG]
    ├── 根据天气检索适合的衣物
    ├── 根据场合检索适配单品
    └── 结合用户偏好排序
    ↓
[步骤4: 搭配生成]
    ├── 生成完整搭配方案
    ├── 使用用户已有衣物
    └── 推荐可购买的新品（可选）
    ↓
[步骤5: 时尚建议输出]
    ├── 文字描述搭配思路
    ├── 色彩协调建议
    ├── 场合适配说明
    └── 购买链接（如有推荐新品）
```

#### 2.5.4 输出格式
```json
{
  "analysis": {
    "detectedItems": ["检测到的单品"],
    "style": "风格分析",
    "colors": "色彩分析"
  },
  "weather": {
    "location": "位置",
    "temperature": "温度",
    "condition": "天气状况",
    "season": "季节"
  },
  "recommendation": {
    "outfitName": "搭配方案名称",
    "items": [
      {
        "clothingId": "衣物ID",
        "name": "衣物名称",
        "reason": "推荐理由"
      }
    ],
    "reasoning": "整体搭配思路",
    "suggestions": "时尚建议"
  },
  "purchaseRecommendations": [
    {
      "name": "推荐商品名",
      "reason": "推荐原因",
      "link": "购买链接"
    }
  ]
}
```

### 2.6 服装视频解析模块

#### 2.6.1 视频上传
- **功能描述**: 支持上传服装相关视频
- **支持格式**: MP4、MOV等常见格式
- **大小限制**: 建议100MB以内

#### 2.6.2 关键帧提取
- **功能描述**: 从视频中提取关键帧进行分析
- **策略**: 每秒1帧或场景切换检测

#### 2.6.3 多维度分析
**分析维度:**
1. **色彩分析**
   - 主色调识别
   - 配色方案评估
   - 色彩和谐度评分
   - 肤色适配性分析

2. **妆容适配度**
   - 妆容风格识别
   - 与服装风格协调性评分
   - 改进建议

3. **气质契合度**
   - 需要先完成气质类型测试
   - 气质类型: 优雅型、自然型、浪漫型、古典型、前卫型、戏剧型等
   - 整体造型与气质匹配评分

4. **搭配合理性**
   - 单品之间协调性
   - 场合适配性

#### 2.6.4 专业评论生成
- **功能描述**: 生成结构化的专业评论
- **输出**: 各维度评分 + 综合评价 + 改进建议

---

### 2.7 面料材质分析模块

#### 2.7.1 面料识别
- **功能描述**: 识别衣物面料材质
- **方式**: AI图像识别（辅助）+ 用户手动输入
- **识别类型**: 棉、麻、丝、羊毛、化纤、混纺、皮革等

#### 2.7.2 面料特性分析
- **功能描述**: 分析面料的各项特性
- **分析维度**:
  - 透气性
  - 保暖性
  - 垂坠感
  - 易皱程度
  - 弹性
  - 耐用性

#### 2.7.3 保养建议
- **功能描述**: 提供专业的面料保养建议
- **内容**:
  - 洗涤方式（手洗/机洗/干洗）
  - 水温要求
  - 熨烫温度
  - 存放要求
  - 注意事项

#### 2.7.4 品质评估
- **功能描述**: 评估面料品质等级
- **评估维度**:
  - 支数（纱线细度）
  - 密度（织物紧密程度）
  - 工艺水平

#### 2.7.5 商户信息导入
- **功能描述**: 支持优质商户衣物信息导入
- **方式**: API对接或批量导入

#### 2.7.6 新品关注
- **功能描述**: 关注品牌新品动态
- **实现**: 爬虫监控 + 消息推送

## 3. 非功能需求

### 3.1 性能需求
- **响应时间**: 普通接口 < 500ms，AI接口 < 3s
- **并发支持**: 支持至少50并发用户
- **图片处理**: 支持图片压缩和缩略图生成

### 3.2 安全需求
- **认证**: JWT Token认证
- **密码**: BCrypt加密存储
- **接口**: 防SQL注入、XSS攻击
- **文件上传**: 类型校验、大小限制

### 3.3 可用性需求
- **可用性**: 99%以上
- **数据备份**: 定期备份
- **错误处理**: 友好的错误提示

### 3.4 扩展性需求
- **模块化设计**: 便于功能扩展
- **配置化**: 关键参数可配置
- **接口版本**: 支持API版本管理

## 5. 接口设计概览

### 5.1 用户模块
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/refresh` - 刷新Token
- `GET /api/user/profile` - 获取用户信息
- `PUT /api/user/profile` - 更新用户信息

### 5.2 衣物模块
- `POST /api/clothings` - 创建衣物
- `GET /api/clothings` - 查询衣物列表
- `GET /api/clothings/{id}` - 获取衣物详情
- `PUT /api/clothings/{id}` - 更新衣物
- `DELETE /api/clothings/{id}` - 删除衣物
- `POST /api/clothings/{id}/recognize` - AI识别衣物
- `PUT /api/clothings/{id}/frequency` - 设置搭配频率

### 5.3 RAG检索模块
- `POST /api/search` - 语义检索衣物
- `POST /api/search/similar` - 查找相似衣物

### 5.4 搭配建议模块
- `POST /api/outfit/advise` - 获取搭配建议
- `POST /api/outfit/analyze` - 分析穿搭照片
- `GET /api/outfit/history` - 获取历史搭配

### 5.5 视频解析模块
- `POST /api/video/upload` - 上传视频
- `GET /api/video/{id}/analysis` - 获取解析结果

### 5.6 风格偏好模块
- `GET /api/style/images` - 获取风格图片
- `POST /api/style/preference` - 标记偏好
- `GET /api/style/recommendations` - 获取推荐
