# 基于Spring Boot + AI Agent的穿搭推荐系统技术架构文档

## 一、系统架构设计

### 1.1 整体架构

本系统采用基于Spring Boot的微服务架构，整体架构分为四个层次：接入层、业务层、AI层和数据层。

**接入层**：负责处理外部请求，包括API网关、负载均衡、认证授权等组件。
**业务层**：承载系统的核心业务逻辑，包括用户服务、衣橱服务、推荐服务等。
**AI层**：系统的智能化核心，包括大模型适配器、Agent工作流引擎、RAG检索引擎等。
**数据层**：负责数据的持久化存储和访问，包括关系型数据库、向量数据库、对象存储等。

### 1.2 核心技术栈

| 技术类别 | 技术选型 | 版本 | 用途 |

| 后端框架 | Spring Boot | 3.2.0 | 微服务基础框架 |

## 二、模块划分与职责

### 2.1 核心模块

**1. 用户服务 (user-service)**
- 职责：用户注册、登录、信息管理、偏好设置等
- 核心功能：JWT认证、用户画像构建、权限管理
- 技术实现：Spring Security + JWT + Redis

**2. 衣橱服务 (wardrobe-service)**
- 职责：衣物管理、图片上传、搭配频率设置等
- 核心功能：CRUD操作、图片处理、标签管理
- 技术实现：Spring Data JPA + MinIO + 异步任务

**3. AI服务 (ai-service)**
- 职责：大模型调用、Agent工作流执行、RAG检索等
- 核心功能：多厂商模型适配、Prompt模板管理、向量检索
- 技术实现：多模型适配器 + LangChain + Milvus

**4. 推荐服务 (recommendation-service)**
- 职责：智能穿搭推荐、搭配方案生成、时尚建议等
- 核心功能：个性化推荐、场景适配、天气感知
- 技术实现：AI Agent + 规则引擎 + 天气API

**5. 内容服务 (content-service)**
- 职责：视频解析、面料分析、商户信息管理等
- 核心功能：视频处理、图像处理、商户同步
- 技术实现：FFmpeg + OpenCV + 定时任务

### 2.2 公共模块

**1. 公共工具 (common)**
- 职责：提供通用工具类和方法
- 核心功能：日期处理、加密解密、HTTP客户端等
- 技术实现：工具类库

**2. 配置管理 (config)**
- 职责：集中管理系统配置
- 核心功能：环境配置、动态配置、配置加密
- 技术实现：Spring Cloud Config

**3. 数据模型 (model)**
- 职责：定义系统数据模型
- 核心功能：实体类、DTO、VO等
- 技术实现：JPA实体 + Lombok

## 三、核心流程设计

### 3.1 智能推荐流程

1. **请求接收**：用户发送推荐请求，包含场合、风格等参数
2. **意图识别**：Agent工作流引擎解析用户意图
3. **数据准备**：获取用户画像、环境数据（天气、温度）、衣橱数据
4. **检索增强**：RAG系统检索相关服装知识和搭配规则
5. **模型推理**：调用大模型生成个性化搭配方案
6. **结果处理**：格式化输出推荐结果，包含搭配理由和时尚建议
7. **反馈收集**：记录用户对推荐的反馈，用于优化算法

### 3.2 视频解析流程

1. **视频上传**：用户上传穿搭视频
2. **智能抽帧**：提取视频中的关键帧
3. **多维度分析**：分析色彩、妆容、气质等维度
4. **专业评论生成**：基于分析结果生成专业时尚评论
5. **结果返回**：向用户展示分析结果和评论

### 3.3 面料分析流程

1. **图片上传**：用户上传衣物面料图片
2. **材质识别**：通过图像识别技术识别面料类型
3. **特性分析**：分析面料的物理和化学特性
4. **保养建议生成**：基于面料特性生成保养指南
5. **结果返回**：向用户展示分析结果和建议

### 4.1 关系型数据库

- nickname: VARCHAR(50)
- email: VARCHAR(100) (UK)
- password: VARCHAR(255)
- avatar: VARCHAR(255)
- style_preferences: JSON
- usual_size: VARCHAR(20)
- region: VARCHAR(100)

- last_login_time: DATETIME
- status: INT

**2. 衣物表 (clothing_items)**
- id: BIGINT (PK)
- user_id: BIGINT (FK)
- name: VARCHAR(100)
- brand: VARCHAR(100)
- category: VARCHAR(50)
- color: JSON
- material: VARCHAR(100)
- size: VARCHAR(20)
- purchase_channel: VARCHAR(100)
- purchase_time: DATE
- image_url: VARCHAR(255)
- frequency: INT (1-5)
- privacy_level: VARCHAR(20)
- created_time: DATETIME
- updated_time: DATETIME

**3. 穿搭记录表 (outfit_records)**
- id: BIGINT (PK)
- user_id: BIGINT (FK)
- outfit_date: DATE
- occasion: VARCHAR(50)
- top_id: BIGINT (FK)
- bottom_id: BIGINT (FK)
- shoes_id: BIGINT (FK)
- accessories: JSON
- user_rating: INT (1-5)
- notes: TEXT
- created_time: DATETIME

**4. 搭配关系表 (matching_pairs)**
- id: BIGINT (PK)
- top_id: BIGINT (FK)
- bottom_id: BIGINT (FK)
- matching_score: DOUBLE
- occasion_tags: JSON
- description: TEXT
- created_time: DATETIME

**5. 知识文档表 (knowledge_documents)**
- id: BIGINT (PK)
- document_type: VARCHAR(50)
- title: VARCHAR(255)
- content: TEXT
- source_url: VARCHAR(255)
- author: VARCHAR(100)
- publish_date: DATE
- tags: JSON
- vector_id: VARCHAR(100)
- created_time: DATETIME

### 4.2 向量数据库

**Milvus 集合设计**
- 集合名称：outfit_knowledge
- 字段：
  - id: VARCHAR(100) (PK)
  - embedding: FLOAT_VECTOR(768) (向量字段)
  - document_id: VARCHAR(100) (关联文档ID)
  - document_type: VARCHAR(50) (文档类型)
  - content: VARCHAR(512) (文本内容)
  - tags: VARCHAR(256) (标签)
  - created_time: TIMESTAMP (创建时间)

**索引类型**：IVF_FLAT + HNSW
**向量维度**：768 (Sentence-BERT)
**相似度度量**：余弦相似度

## 五、API设计

### 5.1 接口分类

| 接口模块 | 路径前缀 | 主要功能 | 权限要求 |
|----------|----------|----------|----------|
| 用户管理 | /api/v1/users | 注册、登录、信息管理 | 公开/认证 |
| 衣橱管理 | /api/v1/wardrobe | 衣物CRUD、图片上传 | 认证 |
| 推荐服务 | /api/v1/recommendations | 智能推荐、搭配分析 | 认证 |
| 视频分析 | /api/v1/video | 视频上传、分析结果 | 认证 |
| 知识库 | /api/v1/knowledge | 知识搜索、面料信息 | 认证 |
| 商户管理 | /api/v1/merchants | 品牌关注、新品动态 | 认证 |
| 面料分析 | /api/v1/fabric | 面料识别、特性分析 | 认证 |

### 5.2 核心接口示例

**1. 用户登录**
- URL: POST /api/v1/users/login
- 请求体: {"email": "user@example.com", "password": "password123"}
- 响应: {"code": "200", "message": "登录成功", "data": {"token": "...", "user": {...}}}

**2. 添加衣物**
- URL: POST /api/v1/wardrobe/items
- 请求体: {"name": "T恤", "brand": "优衣库", "category": "上衣", ...}
- 响应: {"code": "200", "message": "添加成功", "data": {"itemId": 1, ...}}

**3. 获取推荐**
- URL: GET /api/v1/recommendations?occasion=daily&weather=sunny
- 响应: {"code": "200", "message": "获取成功", "data": {"recommendations": [...]}}

**4. 分析视频**
- URL: POST /api/v1/video/analyze
- 请求体: multipart/form-data (video文件)
- 响应: {"code": "200", "message": "分析成功", "data": {"videoId": "...", "status": "analyzing"}}
