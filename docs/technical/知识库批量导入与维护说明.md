# 知识库批量导入与维护说明

这份文档对应当前管理端“知识库管理”页面，适用于维护以下三类数据：

- 面料知识
- 洗护知识
- 穿搭指南

## 入口

管理端登录后，进入：

- `知识库管理`

页面里现在支持：

- 列表查询
- 新增 / 编辑
- 删除
- 当前条目预览
- 批量导入 JSON 数组

## 批量导入规则

每次导入只处理当前 tab 对应的一种类型：

- `面料`
- `指南`
- `洗护`

导入区支持直接粘贴 `JSON 数组`，例如：

```json
[
  {
    "name": "棉麻混纺",
    "alias": "cotton linen blend,棉麻",
    "fabricType": "混纺面料",
    "warmthScore": 48,
    "breathabilityScore": 88,
    "comfortScore": 82,
    "durabilityScore": 70,
    "summary": "兼顾棉的舒适与亚麻的清爽",
    "properties": "透气、自然纹理明显，但较易起皱。",
    "careGuide": "建议轻柔机洗或手洗，阴干。",
    "suitableSeasons": "春,夏",
    "suitableOccasions": "日常,通勤,度假",
    "keywords": "棉麻,透气,夏季"
  }
]
```

页面中有两个辅助动作：

- `填入模板`
- `清空导入区`

## 更新策略

导入面板有一个开关：

- `遇到同名 / 同编码时自动更新旧数据`

规则如下：

- 面料：按 `name` 判断是否已存在
- 指南：按 `title` 判断是否已存在
- 洗护：按 `symbolCode` 判断是否已存在

如果开关开启：

- 已存在则更新
- 不存在则新增

如果开关关闭：

- 已存在则跳过
- 不存在则新增

导入完成后，页面会弹出统计结果：

- 新增多少条
- 更新多少条
- 跳过多少条

## 接口

当前后台新增了 3 组批量导入接口：

- `POST /api/v1/admin/knowledge/fabrics/import`
- `POST /api/v1/admin/knowledge/guides/import`
- `POST /api/v1/admin/knowledge/care-labels/import`

请求体结构统一为：

```json
{
  "updateExisting": true,
  "items": []
}
```

## Demo 数据脚本

如果你不是人工逐条维护，而是想直接导入仓库内置的演示知识库和演示账号，直接运行：

```powershell
.\backend\manual-tests\prepare-demo-data.ps1 -Password 你的MySQL密码
```

对应 SQL 文件：

- `backend/manual-tests/seed-demo-data.sql`

这个脚本适合：

- 新环境初始化
- 答辩前快速恢复演示数据
- 联调时重建基础知识库
