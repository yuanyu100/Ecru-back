# 基于Spring Boot + AI Agent的穿搭推荐系统App端设计文档

## 一、视觉基调 (Visual Identity V2)

### 1.1 核心关键词
- 禅意 (Zen)
- 静谧 (Quiet)
- 温暖 (Warmth)
- 无干扰 (Distraction-free)

### 1.2 色彩体系

| 色彩类型 | 色值 | 描述 | 用途 |
|----------|------|------|------|
| 背景色 | #F5F5DC 或 #FAFAF5 | 暖米色，像未漂白的棉麻 | 全局背景 |
| 文字色 | #333333 | 深炭灰，比纯黑柔和 | 功能文字 |
| 点缀色 | #7799BA 或 #D4A5A5 | 莫兰迪蓝或干燥玫瑰粉 | 交互反馈（如点赞） |

### 1.3 字体设计
- **大标题**：衬线体（Songti/Source Han Serif），营造文学感
- **功能文字**：无衬线体，保证可读性
- **字号层次**：标题 20-24px，正文 16px，辅助文字 14px
- **字重**：标题 medium，正文 regular

## 二、页面设计

### 2.1 页面一：首页「空」 (Home - The Blank Canvas)

**定位**：极致留白，唯一的焦点是"对话与灵感"。无顶部栏，无多余图标。

#### 2.1.1 顶部区域 (Top Area)
- **极致简化**：砍掉天气胶囊、砍掉消息铃铛
- **左上角**：一个极小的、极简线条的「汉堡菜单」图标 (三条短线) 或者用户的微缩头像
  - **交互**：点击后，屏幕左侧优雅地滑出一个半透明的磨砂玻璃层（抽屉），里面藏着"我的衣橱"、"设置"、"切换主题（米黄/黑白/莫兰迪）"等所有功能入口。平时完全隐藏，不打扰视线。

#### 2.1.2 核心交互区 (Center Stage - The Prompt)
- **位置**：屏幕视觉中心偏上（黄金分割点）
- **形态**：只有一行温柔的文字（打字机效果出现）：
  - "今天想穿成什么样？" 或者 "上海 18°C，微风。"（仅仅是淡淡的提示）
- **输入框**：文字下方是一个极其简单的下划线或透明胶囊框
  - **Placehoder**："输入如：去见前任、蓝色系、优衣库..."
  - **交互**：用户点击输入框，键盘弹起。输入需求后，按回车，AI 开始思考。

#### 2.1.3 推荐流区域 (The Flow - Below the Fold)
- **默认状态**：屏幕下半部分是大面积留白，或者是 1-2 张极具美感的**"今日精选"**穿搭图（拍立得风格），安静地躺在那里。
- **交互逻辑**：
  - **喜欢 (Like)**：双击图片，图片中心出现一个淡雅的红心动效，随后红心缩小飞入左上角菜单（代表已收藏）。
  - **不喜欢 (Dislike)**：长按图片，图片像烟雾一样消散（Fade out），下方自动浮现一张新的推荐。
  - **折叠/展开**：屏幕底部有一个极小的**"⌄" (向下箭头)** 或 "..."
    - 用户上滑屏幕，或是点击箭头，更多推荐才会像瀑布一样流出来。
    - 用户下滑，推荐收起，回归到只有输入框的"极简模式"。

### 2.2 页面二：侧边抽屉「藏」 (The Drawer)

**定位**：把所有繁琐的功能都藏在这里，保持首页干净。

- **唤起方式**：点击首页左上角图标
- **菜单内容**：纯文字列表，字体稍大，间距宽松
  1. 我的衣橱 (My Wardrobe) -> 跳转至衣橱管理页
  2. 收藏灵感 (Saved Looks) -> 查看点赞过的穿搭
  3. 面料百科 (Fabric Wiki) -> 简单的知识库
  4. 主题设置 (Theme) -> 包含：米黄（默认）、冷灰、暗夜模式
  5. 关于我们 (About)

### 2.3 页面三：衣橱管理「序」 (Wardrobe - Order)

**定位**：虽然是工具页，也要保持美感。

- **布局**：依然是米黄色背景
- **展示**：衣服不再是密密麻麻的方块，而是像挂在衣架上一样
  - **列表模式**：左边是衣服缩略图（去底），右边是衣服名称和简单的 Tag（#棉 #白色）
  - **添加按钮**：页面右下角，一个极细的圆形线条包围的 "+"。点击后优雅展开"拍照/相册"选项。

### 2.4 页面四：AI 响应页「答」 (The Answer)

**定位**：当用户在首页输入需求后（例如"想穿蓝色色系"），跳转或变化至此状态。

- **过渡动画**：输入框文字上移，渐隐。屏幕中央浮现 AI 的回答。
- **呈现形式**：
  - **文字建议**："蓝色很衬今天的晴天。试试这套深蓝针织衫搭配浅色牛仔裤。"（字体优雅，像便签）。
  - **视觉方案**：文字下方并排展示 2-3 套方案（图片）。
    - 方案 A：主图
    - 方案 B：次选图（稍微小一点）
  - **操作**：只有简单的两个按钮（线性图标）："采用" (Check) 和 "换一换" (Refresh)。

## 三、核心体验优化点 (UX Polish)

### 3.1 零打扰 (Zero Disturbance)
- 没有任何红点气泡
- 除了用户主动唤起，App 永远保持安静
- 通知默认关闭，用户可在设置中开启

### 3.2 触感反馈 (Haptics)
- **双击点赞**：手机给予一个极轻微的震动（Taptic Engine），模拟心跳的感觉
- **长按消除**：当长按消除不喜欢图片时，给予一个逐渐减弱的震动，模拟消散感
- **按钮点击**：所有按钮点击都有轻微的触感反馈

### 3.3 动态背景 (Subtle Motion)
- **质感纹理**：米黄色背景不是死板的纯色，可以增加一点点噪点纹理 (Grain)，像纸张一样
- **环境感知**：如果外面下雨，背景上可以有极淡的、几乎看不见的水痕划过（通过天气 API 触发，但非常隐晦）
- **呼吸效果**：输入框和按钮可以有极轻微的呼吸动画，增加生命力

### 3.4 交互动效
- **打字机效果**：首页提示文字的出现采用打字机效果
- **平滑过渡**：页面间的跳转和元素的出现/消失都采用平滑的过渡动画
- **微交互**：所有用户操作都有相应的微交互反馈

### 3.5 响应式设计
- **适配不同尺寸**：从 iPhone SE 到 iPhone Pro Max 都能保持良好的视觉效果
- **横竖屏适配**：支持横屏模式，特别是在查看穿搭图片时

## 四、技术实现细节

### 4.1 动画实现
- **Flutter**：使用 Flutter 的 AnimationController 和 Tween 实现平滑动画
- **React Native**：使用 Animated API 或 Lottie 实现复杂动画
- **CSS**：如果使用 Web 技术，使用 CSS animations 和 transitions

### 4.2 性能优化
- **图片懒加载**：推荐图片采用懒加载策略
- **内存管理**：及时释放不需要的资源
- **动画性能**：使用硬件加速，避免卡顿

### 4.3 无障碍设计
- **屏幕阅读器支持**：为所有元素添加适当的 aria-label
- **高对比度模式**：支持系统级高对比度设置
- **字体大小调整**：支持系统级字体大小设置

## 五、用户场景示例

### 5.1 早晨穿搭决策
1. 用户打开 App，看到首页的提示文字
2. 点击输入框，输入"今天上班想穿得专业一点"
3. AI 响应页出现，提供 2-3 套专业风格的穿搭方案
4. 用户双击喜欢的方案，方案被收藏
5. 用户点击"采用"按钮，完成决策

### 5.2 衣橱管理
1. 用户点击首页左上角的汉堡菜单
2. 侧边抽屉滑出，点击"我的衣橱"
3. 进入衣橱管理页面，查看已有的衣物
4. 点击右下角的"+"按钮，选择"拍照"
5. 拍摄新衣物，系统自动识别并添加到衣橱

### 5.3 灵感探索
1. 用户打开 App，看到首页的"今日精选"穿搭图
2. 上滑屏幕，查看更多推荐
3. 双击喜欢的穿搭，添加到收藏
4. 长按不喜欢的穿搭，让其消失并显示新推荐
5. 点击首页输入框，输入"周末约会穿搭"
6. AI 提供约会风格的穿搭方案

## 六、设计原则

1. **极简主义**：少即是多，去除所有不必要的元素
2. **用户中心**：以用户的需求和体验为中心
3. **美学价值**：注重视觉美感和设计细节
4. **功能完整性**：虽然简约，但功能完整
5. **技术可行性**：设计方案要考虑技术实现的可行性

## 七、结语

本设计文档详细描述了基于Spring Boot + AI Agent的穿搭推荐系统App端的设计方案。通过极致的极简主义设计、温暖的视觉基调、流畅的交互体验，旨在为用户提供一个专注、优雅、智能的穿搭推荐工具。

设计的核心是"空"与"藏"的哲学，通过留白和隐藏，让用户的注意力集中在最核心的功能——与AI的对话和穿搭灵感的获取上。同时，通过精心的动画和反馈，为用户带来愉悦的使用体验。

这种设计风格不仅符合现代用户对简洁美学的追求，也为AI Agent的智能推荐提供了一个理想的展示舞台，让科技与美学完美融合。
