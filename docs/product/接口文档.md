# 基于Spring Boot + AI Agent的穿搭推荐系统接口文档

## 一、接口设计原则

系统API遵循RESTful设计规范，采用JSON作为数据交换格式。API设计遵循以下原则：

1. **资源命名清晰**：使用名词表示资源，动词表示操作，避免使用复杂的URL路径
2. **HTTP动词语义**：正确使用GET、POST、PUT、DELETE等HTTP方法
3. **版本控制**：通过URL路径中的v1、v2等进行版本管理
4. **错误处理统一**：采用统一的错误响应格式，包含错误码、错误信息等
5. **认证授权安全**：所有接口（除登录注册外）均需JWT令牌认证

## 二、基础结构

### 2.1 响应格式

所有API响应均采用统一格式：

```json
{
  "code": "200",
  "message": "操作成功",
  "data": {...},
  "timestamp": "2026-01-29T12:00:00Z"
}
```

**错误响应格式**：

```json
{
  "code": "400",
  "message": "参数错误",
  "error": "具体错误信息",
  "timestamp": "2026-01-29T12:00:00Z"
}
```

### 2.2 认证方式

采用JWT令牌认证，在请求头中添加：

```
Authorization: Bearer {token}
```

## 三、错误码定义

| 错误码 | 描述 | HTTP状态码 |
|--------|------|------------|
| 200 | 操作成功 | 200 |
| 400 | 参数错误 | 400 |
| 401 | 未授权，令牌无效或过期 | 401 |
| 403 | 禁止访问，权限不足 | 403 |
| 404 | 资源不存在 | 404 |
| 409 | 资源冲突，如邮箱已存在 | 409 |
| 500 | 服务器内部错误 | 500 |
| 502 | 外部服务错误，如AI模型调用失败 | 502 |
| 503 | 服务暂时不可用 | 503 |

## 四、认证与授权

### 4.1 JWT令牌

- **获取方式**：通过登录接口获取
- **有效期**：24小时
- **刷新机制**：支持令牌刷新，通过`/api/v1/users/refresh-token`接口
- **使用方式**：在请求头中添加`Authorization: Bearer {token}`

### 4.2 权限控制

| 接口类型 | 权限要求 |
|----------|----------|
| 用户管理 | 仅本人可操作自己的账户 |
| 衣橱管理 | 仅本人可操作自己的衣橱 |
| 推荐服务 | 所有登录用户均可使用 |
| 知识库 | 所有登录用户均可访问 |
| 商户管理 | 管理员权限 |

## 五、接口安全

1. **HTTPS传输**：所有接口均采用HTTPS协议
2. **密码加密**：用户密码采用BCrypt加密存储
3. **敏感数据保护**：用户隐私数据传输采用AES加密
4. **请求限流**：每用户每秒最多100次请求
5. **防止SQL注入**：使用参数化查询
6. **防止XSS攻击**：输入验证和输出转义
7. **防止CSRF攻击**：使用CSRF令牌
